Description:

 

 

Currently, it's not possible to generate more than 64K simultaneous connections ( at least on Linux)

 

This is the max number of ephemeral ports; but it should be possible to bypass this limit with multiple IPs ; however this doesn't work because the bind lookup algorithm of the linux kernel doesn't take into account the local IP.

 

more info on this topic:

http://aleccolocco.blogspot.com/2008/11/ephemeral-ports-problem-and-solution.html

(since erlang always use the bind system call, the limitation will exist also if several servers/ports are used)

 

So one way to bypass this limitation is to have our own port allocation algorithm
